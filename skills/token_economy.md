# Token Economy & Conciseness

- Be concise. Do not explain what you are about to do, just do it.
- Do not write "Here is the code:" or "I have implemented this for you". Just output the code.
- If the change is small (less than 10 lines), output only the code block without surrounding text.
- Do not repeat code that hasn't changed. Use `// ... existing code ...` to indicate unchanged parts.
- Do not list "Next steps" or "Suggestions" unless I explicitly ask for them.
- Do not add comments explaining obvious logic (e.g., `// add 2+2`).
